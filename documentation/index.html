1. Getting Started

Thank you for purchasing Elara Shop. Follow the four steps below to get the template running locally.

  1. Unzip the archive you downloaded from ThemeForest.
  2. Open a terminal in the project root.
  3. Install dependencies with npm install (or yarn / pnpm install).
  4. Start the dev server with npm run dev and open http://localhost:3000.

6. Customizing the Template

Brand colors

All design tokens are defined in src/app/globals.css inside the @theme block. Change a value once and it propagates through every component.

@theme {
  --color-primary: #000000;
  --color-accent: #1c558b;
  --color-accent-dark: #15406a;
  --color-success: #02d071;
  /* ...surface, text, and border tokens */
}

Logo

The header and footer logo currently use a colored block with the letter E. Replace the markup in src/components/layout/Header.tsx and src/components/layout/Footer.tsx with your SVG or image.

Typography

The template uses Inter from Google Fonts (loaded via next/font). To change it, edit the import at the top of src/app/layout.tsx.

Site metadata & SEO

Site-wide metadata lives in src/app/layout.tsx. Per-page metadata can be added by exporting metadata (or generateMetadata) from each page.tsx.

7. Replacing Demo Data

All demo content — products, categories, brands, hero slides, mega-menu items — lives in src/data/mock.ts. Update this file or wire it to your CMS / API and the UI will follow.

Tip: the existing TypeScript interfaces in mock.ts describe the exact shape every component expects. Keeping the same shape when you connect a real backend means zero component changes.

8. Dark / Light Mode

Dark mode is implemented using Tailwind 4's @custom-variant dark. The ThemeProvider in src/context/ThemeContext.tsx manages state, persists the choice to localStorage, and falls back to the system preference on first visit.

Color overrides for dark mode are defined in src/app/globals.css under the .dark { … } block.

9. Cart & Wishlist

Both features use the same architecture: a React Context provider wraps the app in layout.tsx, and any component reads state via the useCart() / useWishlist() hooks. State is persisted to localStorage.

Cart API

MethodDescription
addToCart(product, qty?)Add a product or increment quantity.
removeFromCart(id)Remove a product entirely.
updateQuantity(id, qty)Set an exact quantity.
clearCart()Empty the cart.
isInCart(id)Returns true if present.
totalItems / totalPriceComputed totals.

Wishlist API

MethodDescription
toggleWishlist(product)Adds if missing, removes if present.
removeFromWishlist(id)Remove a product.
isInWishlist(id)Membership check.
totalItemsItem count.

10. Deployment

Elara Shop is configured for static export (output: "export" in next.config.ts). Running npm run build produces a fully static site in /out that can be hosted anywhere.

  • Firebase Hosting: npm run deploy uses the included firebase.json.
  • Vercel / Netlify: connect the repository and deploy with default Next.js settings.
  • Any static host: upload the contents of /out to S3, Cloudflare Pages, GitHub Pages, Nginx, etc.
If you need server-rendered features (e.g. a checkout API route), remove output: "export" from next.config.ts and deploy to a Node-capable host like Vercel.
